# 🚢 타이타닉 생존 예측: 로지스틱 회귀 심층 분석

## 1. 프로젝트 개요

이 프로젝트는 고전적인 데이터셋인 **타이타닉 생존 예측**을 주제로, 머신러닝의 기본 모델인 **로지스틱 회귀(Logistic Regression)**를 **직접 구축하고 평가**하는 과정을 심층적으로 다루었습니다.

**목표:**

1.  **알고리즘 원리 이해:** NumPy를 활용하여 모델의 핵심 수학적 원리(손실 함수, 경사 하강법)를 직접 구현하여 작동 방식을 근본적으로 이해합니다.
2.  **표준 파이프라인 구축:** Scikit-learn을 사용한 효율적인 파이프라인을 구축하고, 모델 성능을 체계적으로 평가합니다.

## 3. 데이터 전처리 과정 요약

* **불필요 컬럼 제거:** 예측과 관련성이 낮거나 중복되는 컬럼(`adult_male`, `deck`, `alive`, `alone` 등)을 제거하여 모델 학습의 노이즈를 줄였습니다.
* **결측치 처리 (Imputation):**
    * `Age` (나이)의 결측치는 데이터의 **중앙값(Median)**으로 대체했습니다.
    * `Embarked` (승선지)의 결측치는 가장 빈도가 높은 값인 **최빈값(Mode)**으로 대체했습니다.
    * `Cabin` (객실 번호)은 결측치가 너무 많아 컬럼 자체를 제거했습니다.
* **범주형 변수 인코딩:** 문자열 형태의 범주형 변수(`Sex`, `Embarked`)를 로지스틱 회귀 모델이 처리할 수 있는 **숫자 형태**로 변환했습니다 (원-핫 인코딩 또는 레이블 인코딩).

---

## 4. 모델 원리: NumPy 기반 구현

로지스틱 회귀의 동작 원리를 이해하기 위해 NumPy만을 사용해 모델을 구현하고, **경사 하강법(Gradient Descent)**을 통해 최적의 가중치를 찾는 과정을 시각적으로 확인했습니다.

### 4.1. 핵심 구성 요소

| 요소 | 수식 (개념) | 역할 |
| :--- | :--- | :--- |
| **가설 함수** | 시그모이드 함수: $\frac{1}{1 + e^{-Z}}$ | 선형 결과를 0과 1 사이의 **확률**로 변환 |
| **손실 함수** | 이진 교차 엔트로피 | 예측된 확률과 실제 값 사이의 **오차** 측정 |
| **최적화** | 경사 하강법 | 손실 함수의 **기울기**를 따라 가중치를 업데이트하여 오차를 최소화 |
---

## 5. 최종 결과 및 해석 (Scikit-learn)

전처리된 데이터를 훈련/테스트 세트로 분리하고, Scikit-learn의 `LogisticRegression`을 사용하여 얻은 최종 성능 평가 결과입니다.

### 5.1. 평가 지표

| 지표 | 값 (예시) | 의미 |
| :--- | :--- | :--- |
| **정확도(Accuracy)** | 0.8201 | 전체 예측의 정답률입니다. |
| **정밀도(Precision)** | 0.8261 | 모델이 '생존'이라 예측했을 때 실제 생존한 비율입니다. |
| **재현율(Recall)** | 0.7215 | 실제 생존자 중에서 모델이 생존자로 올바르게 찾아낸 비율입니다. |
| **F1-Score** | 0.7780 | 정밀도와 재현율의 균형을 나타냅니다. |

### 5.2. 결과 해석: 재현율(Recall)의 중요성

타이타닉 프로젝트는 생존(긍정, Positive)을 예측하는 문제입니다. 이 경우 **실제 생존자를 놓치는 것(False Negative, FN)**이 **사망자를 생존자로 잘못 예측하는 것(False Positive, FP)**보다 더 심각한 오류로 간주됩니다.

따라서, **실제 생존할 사람을 최대한 많이 예측**하는 능력을 나타내는 **재현율(Recall)** 지표를 높게 유지하는 것이 중요하다고 해석할 수 있습니다. 현재 모델은 실제 생존자 중 약 72%를 성공적으로 예측했습니다.

## 6. 개선 방향 및 자기 고찰

### 6.1. 내가 배운 것 (고찰)

이 프로젝트를 통해 단순히 라이브러리 함수를 사용하는 것을 넘어, 머신러닝 알고리즘의 **핵심 작동 원리(경사 하강법)**를 NumPy 코드로 직접 구현해 봄으로써 모델 학습 과정에 대한 **근본적인 이해**를 다질 수 있었습니다.

### 6.2. 향후 개선 방향 (현재 학습 수준)

1.  **특성 공학(Feature Engineering) 적용:**
    * `SibSp`와 `Parch` 컬럼을 조합하여 **가족 크기(`FamilySize`)**라는 새로운 특성을 만들고, 이것이 생존에 미치는 영향을 추가로 분석하여 모델 성능을 개선할 수 있습니다.
2.  **규제(Regularization) 튜닝:**
    * 로지스틱 회귀 모델의 **규제 강도(`C`)**를 조정하여 과적합(Overfitting)을 방지하고 일반화 성능을 높이는 하이퍼파라미터 튜닝을 시도해 볼 것입니다.
3.  **다음 단계 도전:**
    * 선형 모델인 로지스틱 회귀를 완성했으므로, 다음 단계로 **결정 트리(Decision Tree)**나 **랜덤 포레스트(Random Forest)**와 같은 **비선형 모델**을 적용하여 성능을 비교 분석하는 것에 도전할 계획입니다.
